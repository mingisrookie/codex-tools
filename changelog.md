## 更新日志

- v2.0.2 (2026-06-13)
  1. API 反代启动时动态解析本机官方 Codex CLI 版本，并让上游 `Version`、`User-Agent: codex_cli_rs/<version>` 和模型 catalog `client_version` 跟随该版本；无 CLI 环境可用 `CODEX_TOOLS_CODEX_CLIENT_VERSION` 显式覆盖。
  2. 清理反代 HTTP 客户端的自定义特征：移除 `codex-tools-proxy/0.1` UA、强制 HTTP/1、禁用 gzip 和禁用连接池等设置，改回项目 reqwest/rustls 默认行为。
  3. 修复单账号或全部候选都处于 runtime cooldown 时，本地候选池被清空并误报“全部代理账号 5 小时用量已耗尽或暂不可用”的问题；runtime cooldown 现在只在仍有其他候选账号时用于失败降权，不能作为终止条件。
  4. 增强 `/v1/responses` 兼容性：支持 `input` 字符串写法，并剥离 Codex upstream 不接受的 `max_output_tokens` 字段。
  5. 补齐远程 proxyd 打包源码清单，并在 Linux proxyd 文档中说明远程机器缺少 Codex CLI 时的版本覆盖方式。

- v2.0.1 (2026-05-27)
  1. 修复 `app-new.exe` 在请求携带已删除或不在启用池的 `ChatGPT-Account-Id` 时清空候选池，误报“全部代理账号 5 小时用量已耗尽或暂不可用”的问题。
  2. 指定账号缺失时改为保留可用候选池继续负载均衡，并允许顺序目标更新到实际成功账号，避免持续黏在失效账号。
  3. 调整 average 负载均衡排序：优先按账号实际用量选择，free/plus 仅作为同用量下的排序因素，避免有低用量 Plus 时反复先打高用量 free。
  4. 补充 stale 指定账号、顺序目标持久化和 average 排序回归测试，并按 Tauri 项目构建链路验证发布产物。

- v2.0.0 (2026-05-26)
  1. 版本统一升到 2.0.0，覆盖前端 package、Tauri 配置、主程序 Cargo、proxyd Cargo 与 lockfile。
  2. 代理稳定性：修复上游响应头长时间无返回后 cooldown 时间戳过旧的问题，并保留顺序模式当前账号后台持久化与账号未变化跳过落盘。
  3. 账号路由：继续使用候选池缓存、严格 `ChatGPT-Account-Id` 限定、5 小时用量过滤与失败后健康账号续跑，降低慢首字和误回退。
  4. 授权兜底：当前 Codex auth 缺少 refresh token 时不再覆盖账号池完整凭据，并支持从 profile auth 自动修复缺失 refresh token、清理错误 blocked 状态。
  5. 用量与界面：账号用量自动刷新默认 1 分钟，并支持在账号界面按分钟调整；顶部栏/设置/i18n 同步展示与保存该配置。
  6. 文档与统计：补充 `docs/api-proxy.md` 的 cooldown、顺序持久化和 refresh token 兜底说明；修正 dashboard in-flight 回归测试对 0ms 和全局空队列的并发假设；本次 v2.0.0 相对 v1.9.0 共 27 个文件、579 行新增、52 行删除。

- v1.9.1 (2026-05-26)
  1. 修复上游响应头长时间无返回后 cooldown 时间戳过旧，导致候选账号冷却立即失效的问题
  2. 顺序模式下当前账号持久化改为后台写入，并在账号未变化时跳过落盘，避免本地 `accounts.json` / `store_lock` 阻塞首字路径
  3. 账号用量接口自动刷新默认改为 1 分钟一轮，并支持在账号界面按分钟调整
  4. 修复当前 Codex auth 缺少 refresh token 时覆盖已存账号凭据的问题，并支持从 profile auth 修复缺失的 refresh token
- v1.9.0
  1. 修复 Codex Tools 直连比账号 CLI 慢的问题，去掉指定账号请求中的 sequential target 持久化写入延迟
  2. 增加账号候选池缓存、严格 `ChatGPT-Account-Id` 限定和 5 小时用量过滤，避免反复读取账号文件或回退到错误账号
  3. 顺序模式改为“当前账号优先，失败后继续尝试其它健康账号”，避免未标记账号请求在当前账号传输失败后误报全部账号耗尽
  4. 将 `send_failed` cooldown 收敛到 30 秒，并对 `error sending request for url` 增加一次快速重试
  5. 切换 Codex 上游 HTTP 客户端到 rustls TLS 栈，并补充 900k fast/xhigh 对比 benchmark 脚本和回归测试
