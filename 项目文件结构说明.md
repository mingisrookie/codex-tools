# 项目文件结构说明

本文档维护 Codex Tools 仓库长期文件结构与职责索引，供 AI 和开发者在动手前快速定位真实代码链路。

更新规则：

- 新增、删除、重命名长期维护文件或目录后，必须同步更新本文件。
- 文件职责发生明显变化时，必须更新对应说明。
- 静态图标、截图、锁文件、构建产物不逐个展开；按目录或配置说明即可。
- `node_modules/`、`dist/`、`src-tauri/target/`、`src-tauri/gen/`、日志、临时输出和本地密钥目录不属于长期结构说明范围。

## 1. 根目录

- `AGENTS.md`：仓库级 Codex/AI 执行入口，要求开发前阅读三份根目录长期文档，并约束图片生成走本地 API proxy。
- `README.md`：项目对外入口，包含徽章、项目状态、快速导航、应用截图、主要能力、文档入口、开发和发布说明。
- `how to use.md`：面向使用者的操作说明，包括导入账号、查看用量与重置卡到期时间、切换账号、开启 API 反代。
- `changelog.md`：版本更新记录。
- `bounty-radar-latest.md`：历史 bounty 扫描快照/本地工作记录，不属于核心产品文档；更新或提交前要确认仍在本次任务范围内。
- `LICENSE`：MIT License。
- `package.json`：前端/Tauri 开发脚本与 npm 依赖定义。当前主要脚本为 `dev`、`dev:desktop`、`build`、`lint`、`preview`、`tauri`。
- `package-lock.json`：npm 锁文件。
- `vite.config.ts`：Vite 配置。
- `eslint.config.js`：ESLint 配置。
- `tsconfig.json`、`tsconfig.app.json`、`tsconfig.node.json`：TypeScript 配置。
- `index.html`：Vite HTML 入口。
- `qa-stubbed-page.yml`：QA / stub 页面相关配置。
- `SKILL.md`：项目附带 skill/设计说明文件。
- `项目文件结构说明.md`：当前文件，维护项目文件结构与职责。
- `项目完整链路说明.md`：从启动、账号、用量、切换、API 反代、cloudflared、远程 proxyd 到发布的完整链路说明。
- `项目开发规范（AI协作）.md`：AI/开发者开发规范、架构边界、测试与文档同步要求。
- `开发者AI开发与PR提交流程.md`：使用 AI 做开发、推送和 PR 的执行流程。

## 2. `.github/`

- `.github/workflows/release.yml`：GitHub Actions 发布流程。当前 tag 或手动触发后使用 Node 22、Rust stable、`npm ci` 和 `tauri-apps/tauri-action` 构建 macOS Apple Silicon、macOS Intel、Windows 包并发布；release body 仍是静态模板，自动从 `changelog.md` 抽取版本段的 patch 需带 `workflow` scope 的 token 后再启用。

## 3. `public/`

- `public/ScreenShot.png`：README 使用的应用截图；公开截图必须使用脱敏示例数据或打码后的真实数据。
- `public/codex-tools.png`、`public/codex-tools-statusbar.png`：公开静态图标资源。

## 4. `scripts/`

- `scripts/dev-preview.ps1`：Windows 本地桌面预览脚本，对应 `npm run dev:desktop`。
- `scripts/bench-900k-fast-xhigh-compare.ps1`：API 反代大上下文 / fast / xhigh 对比 benchmark 脚本。
- `scripts/probe-fast-service-tier.ps1`：fast service tier 探测脚本。

新增脚本必须说明平台、输入、输出、副作用和是否会写入 `output/`。

## 5. `docs/`

- `docs/api-proxy.md`：API 反代详细链路说明，覆盖本地 `/v1` route、鉴权、账号池、请求转换、SSE、失败重试、dashboard、CC Switch 接入和调试命令。
- `docs/linux-proxyd.md`：Linux 独立 proxyd 编译、启动、数据目录、手动运行，以及桌面端远程服务器 SSH 部署、systemd 管理和日志读取方式。
- `docs/assets/`：文档内使用的 SVG/图片资源，例如 API proxy dashboard preview。

API 反代、远程 proxyd 或调试方式变化时，必须同步检查 `docs/`。

## 6. `src/` 前端

- `src/main.tsx`：React 入口，挂载 App 和 i18n provider。
- `src/App.tsx`：应用主布局、tab 切换、顶层组件装配。
- `src/App.css`、`src/index.css`、`src/styles/`：全局样式、组件样式、主题样式、响应式样式。
- `src/types/app.ts`：前端类型定义，与 Tauri/Rust JSON 契约保持同步。
- `src/hooks/useCodexController.ts`：前端核心 controller，负责账号、用量、设置、API 反代、Dashboard、cloudflared、远程 proxyd、更新、导入导出等状态与动作编排。
- `src/hooks/useThemeMode.ts`：主题模式状态。
- `src/components/AccountsGrid.tsx`：账号页主表格、行选择和右侧账号 inspector，负责账号启停、切换、导出、重授权、重命名、删除等入口。
- `src/components/AccountCard.tsx`：旧版账号卡片组件，当前主账号页不再直接挂载，保留作回退参考。
- `src/components/AddAccountDialog.tsx`、`AddAccountSection.tsx`：导入/授权账号入口和弹窗。
- `src/components/ApiProxyPanel.tsx`：API 反代面板，本地端口、key、远程服务器、cloudflared 等入口。
- `src/components/DashboardPanel.tsx`：API 反代 dashboard 与统计展示。
- `src/components/SettingsPanel.tsx`：应用设置。
- `src/components/AppTopBar.tsx`、`BottomDock.tsx`、`MetaStrip.tsx`、`NoticeBanner.tsx`、`UpdateBanner.tsx`、`RemoteDeployProgressToast.tsx`：顶栏、底部导航、摘要条、通知、更新、远程部署进度等 UI。
- `src/components/EditorMultiSelect.tsx`、`SwitchField.tsx`、`ThemeSwitch.tsx`：可复用小组件。
- `src/i18n/`：多语言 provider、catalog、后端错误映射和 locale JSON。
- `src/utils/accountOverview.ts`：账号页顶部统计与 inspector 摘要的派生口径。
- `src/utils/accountRanking.ts`：账号排序/智能切换辅助。
- `src/utils/dashboardAxis.ts`：dashboard 坐标轴/格式化辅助。
- `src/utils/usage.ts`：用量展示格式化辅助。
- `src/constants/externalLinks.ts`：外部链接常量。
- `src/assets/`：前端构建静态资源。

前端改动要优先确认 `useCodexController.ts`、类型、组件、i18n、样式是否都同步。

## 7. `src-tauri/` 桌面后端

- `src-tauri/Cargo.toml`、`Cargo.lock`：Tauri 主程序 Rust crate 配置和锁文件。
- `src-tauri/tauri.conf.json`：Tauri 应用配置，含产品名、版本、窗口、bundle、updater、资源打包。
- `src-tauri/build.rs`：Tauri build script。
- `src-tauri/capabilities/default.json`：Tauri capability 权限配置。
- `src-tauri/icons/`：桌面应用图标资源。
- `src-tauri/src/main.rs`：桌面应用 Rust binary 入口。
- `src-tauri/src/bin/codex-tools-proxyd.rs`：主 crate 内的独立 proxyd binary 入口。
- `src-tauri/src/lib.rs`：Tauri app 装配、命令注册、OAuth callback listener、窗口/托盘/自启动 glue。
- `src-tauri/src/state.rs`：全局运行态 `AppState` 和 API proxy / cloudflared runtime handle。
- `src-tauri/src/models.rs`：前后端共享 JSON 数据结构、默认值、设置 patch、归一化逻辑。
- `src-tauri/src/app_paths.rs`：应用数据目录、Codex 目录等路径解析。
- `src-tauri/src/auth.rs`：Codex/ChatGPT OAuth、token refresh、auth.json 解析与写入。
- `src-tauri/src/account_service.rs`：账号导入、导出、删除、重命名、启停用、用量刷新等业务。
- `src-tauri/src/store.rs`：`accounts.json` 读写、恢复、备份、启动同步当前账号。
- `src-tauri/src/usage.rs`：ChatGPT/Codex 5h、1week、credits 用量抓取，以及 Codex 主动重置卡到期时间读取。
- `src-tauri/src/token_usage.rs`：扫描 Codex session JSONL，统计 token 用量。
- `src-tauri/src/settings_service.rs`：应用设置读取、更新和开机启动同步。
- `src-tauri/src/proxy_service.rs`：本地 Axum OpenAI-compatible API 反代核心。
- `src-tauri/src/proxy_daemon.rs`：无桌面独立 daemon 运行逻辑。
- `src-tauri/src/dashboard_metrics.rs`：API 反代指标记录、窗口统计、in-flight request 管理。
- `src-tauri/src/cloudflared_service.rs`：cloudflared 安装、快速隧道、命名隧道、停止清理。
- `src-tauri/src/remote_service.rs`：远程 Linux proxyd 构建、部署、启动、停止、日志读取。
- `src-tauri/src/editor_apps.rs`：编辑器扫描与重启联动。
- `src-tauri/src/opencode.rs`：Opencode 授权同步与桌面应用联动。
- `src-tauri/src/profile_files.rs`：Codex profile 文件完整性处理。
- `src-tauri/src/cli.rs`：本机 Codex CLI/命令解析辅助。
- `src-tauri/src/i18n.rs`：后端错误和文案本地化辅助。
- `src-tauri/src/tray.rs`：系统托盘菜单和状态展示。
- `src-tauri/src/utils.rs`：通用时间、权限、命令、路径、错误截断等辅助。

Rust 改动要确认 `lib.rs` command 注册、`models.rs` 契约、service 模块、store/runtime 锁和前端类型是否一致。

## 8. `src-tauri/proxyd/`

- `src-tauri/proxyd/Cargo.toml`、`Cargo.lock`：独立 Linux proxyd crate 配置。
- `src-tauri/proxyd/src/main.rs`：独立 daemon 入口，复用主 crate 中的 proxy daemon/service 逻辑。

修改远程或无桌面反代时，必须同步检查主 crate binary、proxyd crate、`docs/linux-proxyd.md` 和 `remote_service.rs` 的打包源文件清单。

## 9. 本地/临时目录说明

- `_bounty_work/`、`output/`、`scan-results/`、`.codex-secrets/` 当前属于本地工作/输出/敏感目录，不应默认纳入普通项目提交。
- `dist/`、`node_modules/`、`src-tauri/target/`、`src-tauri/proxyd/target/` 是构建或依赖产物。
- 新增任何本地输出目录前，先确认是否应写入 `.gitignore`，避免误提交大文件、日志、密钥或生成物。

## DXM 大开发工作流目录

本项目已启用 Trellis/Codex 大开发工作流时，下列目录属于项目级 AI 协作基础设施：

- `.trellis/`：Trellis 项目工作流状态、任务 PRD、spec、workspace journal 和脚本。
- `.trellis/tasks/`：每个开发任务的 `task.json`、`prd.md`、实现上下文和检查上下文。
- `.trellis/spec/`：可复用项目规范；完成任务后应把稳定经验沉淀回这里。
- `.codex/`：项目级 Codex skills、agents、hooks 和配置。
- `.agents/skills/`：跨 agent 共享的 Trellis skill 入口。

维护要求：修改这些目录的事实结构或工作流含义时，同步更新 `AGENTS.md`、`项目开发规范（AI协作）.md` 和本文档。
